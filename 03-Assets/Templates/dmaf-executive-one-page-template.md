# DMAF Executive One-Page Template

## 1. Purpose

The purpose of this template is to create a concise executive-facing one-page summary for a Databricks modernization opportunity, strategy, architecture recommendation, migration roadmap, or delivery initiative.

This template supports:

```text id="gsyq7v"
DMAF-CAP-EXA-006 — Executive Visual Design
```

It is designed to help stakeholders quickly understand:

* what problem is being solved;
* why modernization matters now;
* how Databricks is relevant;
* what the recommended path is;
* what business value is expected;
* what decision or action is required.

---

## 2. Intended Audience

This template is intended for:

* executive sponsors;
* technology executives;
* data and analytics leaders;
* platform owners;
* business stakeholders;
* account executives;
* engagement leads;
* advisory consultants;
* Databricks practice leads.

---

## 3. When to Use This Template

Use this template when:

* an executive summary is needed for a modernization opportunity;
* a proposal needs a concise front-page narrative;
* a discovery outcome must be summarized for decision makers;
* a target-state architecture needs executive framing;
* a migration roadmap needs business-level positioning;
* technical detail must be translated into executive action.

---

# 6. One-Page Visual Layout Guidance

Recommended layout:

```text id="91l3qb"
Top:
- Executive Message
- Decision Requested

Middle:
- Why This Matters Now
- Current-State Challenge
- Target-State Direction

Bottom:
- Business Value
- Recommended Path
- Key Risks / Decisions
- Call to Action
```

Design guidance:

* Keep the final output to one page.
* Use business language first.
* Avoid unnecessary platform detail.
* Use no more than five to seven major bullets.
* Make the decision request visible.
* Use a simple visual flow from current state to target state.
* Include risks only if they influence executive action.
* Avoid dense architecture diagrams unless they are simplified.

# 8. Common Executive Story Patterns

## 8.1 Platform Risk Story

Use when the main driver is legacy risk, deprecation, fragility, or operational exposure.

```text id="rw4zgy"
Modernization is required because the current platform creates growing risk and limits future agility. A Databricks-led target state reduces dependency on legacy tooling while creating a governed foundation for data engineering, analytics, and AI.
```

## 8.2 Business Value Story

Use when the main driver is speed, capability, growth, or value realization.

```text id="wsw36m"
Modernization creates business value by improving how data is delivered, governed, trusted, and reused. Databricks provides the foundation to accelerate delivery while enabling future analytics and AI capabilities.
```

## 8.3 Migration Factory Story

Use when the main driver is a large portfolio migration.

```text id="ntjqeu"
The opportunity should be approached as a migration factory rather than a series of disconnected conversions. A structured wave plan allows the organization to prioritize, sequence, validate, and scale modernization with greater control.
```

## 8.4 Validation and Trust Story

Use when the audience is concerned about correctness, financial accuracy, auditability, or production risk.

```text id="evmd7e"
The modernization path must earn trust through validation. Parallel run, reconciliation, control totals, exception handling, and sign-off criteria should be designed into the program from the beginning.
```
