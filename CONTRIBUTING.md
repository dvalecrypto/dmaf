# Contributing to DMAF

DMAF should be managed like a product, not a one-time project.

## Contribution Rules

Every new artifact should answer:

1. Which DMAF practice domain does it support?
2. Which lifecycle stage uses it?
3. Which guiding principle does it reinforce?
4. Which reusable asset or capability does it improve?
5. Can it be reused across engagements?

## Preferred Formats

- Markdown for documents
- Draw.io for editable diagrams
- PowerPoint for client-facing decks
- Excel for workbooks and calculators
- HTML/CSS/JS for lightweight prototypes
